## 今後身に付けたい技術

### バックエンド（Go / API）

- 個人開発で Go によるAPIサーバー実装（Gin、レイヤード設計、PostgreSQL/Supabase、Cognito、Lambda + API Gateway など）に触れてきたため、今後は **「業務レベルで再現性のある設計・運用」** まで深めたいと考えています。
- 深めたいテーマ：
  - **API設計**：リソース設計、エラーハンドリングの統一、バージョニング、OpenAPI/Swaggerでの仕様管理
  - **認証・認可**：Cognito等を前提に、RBAC/権限境界、トークン運用
  - **非同期処理/バッチ**：集計・通知などをジョブとして分離し、失敗時のリトライや冪等性を担保
  - **DB設計**：正規化とクエリ性能のバランス、インデックス設計、マイグレーション運用
  - **パフォーマンス/信頼性**：ボトルネックの可視化、キャッシュ戦略、レート制御などの設計
  - **テスト**：ユニット/統合テストを前提に、変更に強いAPIの設計と回帰の仕組みづくり

### インフラ（AWS / Terraform）

- Firebase は個人・実務ともに利用してきましたが、個人開発では Terraform によるAWS構築（S3、Cognito、Lambda、API Gateway、DynamoDB、SNS など）にも触れてきました。今後は **IaC前提で運用できるインフラ設計** を深めたいです。
- 深めたいテーマ：
  - **環境分離/運用**：dev/staging/prodの分離、設定管理、リリース手順の標準化
  - **権限設計/セキュリティ**：最小権限、シークレット管理、監査性の担保
  - **観測性**：ログ/メトリクス/アラート設計と、障害対応を回せる仕組み
  - **コスト最適化**：見える化と継続的な改善（無駄なリソースの抑制）

### データ/分析（ログ収集・BigQuery）

- 個人開発でログ収集・分析（Firebase Analytics、BigQuery等）を実装してきたため、今後は **「計測→分析→改善」を継続的に回す設計** を強化したいです。
- 深めたいテーマ：
  - **イベント設計**：KPIに紐づくログ設計、命名規則、スキーマ管理
  - **集計・可視化**：定常レポート化、ダッシュボード、異常検知の仕組み
  - **改善サイクル**：施策→計測→検証の高速化（プロダクト改善に直結させる）

### フロントエンド

#### Web（Next.js / Vue.js）

- 管理画面/オーナー向けコンソールなど **複数フロントを運用する前提** で、中〜大規模開発に必要な設計・運用のノウハウを積みたいです。
- 深めたいテーマ：
  - **構成・状態管理**：ディレクトリ構成、状態管理、APIクライアント設計の標準化
  - **BFF/API Routes**：フロントに最適化したAPI（認証・権限・キャッシュ）設計
  - **レンダリング戦略**：SSR/SSG/ISRの使い分けとパフォーマンス最適化
  - **運用**：エラー監視、ログ、権限周りの保守性

#### Flutter

- Flutter での開発経験を活かしつつ、プロダクト品質を高めるための習慣や仕組みづくりを強化していきたいです。
  - **E2Eテスト**：手動テスト削減・リリースサイクル高速化
  - **Widget Book**：UIコンポーネントのカタログ化・品質の平準化
  - **Crashlytics活用**：レポート分析〜改善サイクルの仕組み化

#### ネイティブ開発・プラグイン開発（Swift / Kotlin）

- 実務でAndroid TVアプリ開発を経験したことをきっかけに、Flutterの可能性を広げるため、ネイティブ開発にも挑戦したいと考えています。個人開発ではまだ経験が少ないため、今後実務で深めていきたいです。
- 深めたいテーマ：
  - **Flutterプラグイン開発**：Swift/Kotlinによるネイティブ機能の実装と、Flutter側への橋渡し
  - **デバイス拡張**：Apple Watch / Vision Pro / CarPlay など、モバイル以外のデバイスでの開発
